**Notizen**
JavaScript wird dazu verwendet eine interaktive Webseite zu erstellen.

Heutzutage ist sie immer noch die einzige Sprache die in jedem Browser verwendet werden kann.

NPM steht für Node Package Manager. Es ist der Paketmanager für node.js um Software-Pakete zur Entwicklung von Webanwendungen zu verwalten.

Drei bekannte Schlüsselwörter, um Variablen zu deklarieren, sind let, var & const.
-let wird nur in einem Block gespeichert. Das bedeutet man kann ausserhalb dieses Blocks nicht auf diese Variable zugreifen.

-var wird nur innerhalb der ganzen Funktion gespeichert. Das bedeutet man kann innerhalb der Funktion und allen Blöcken in dieser Funktion auf die Variable zugreifen.

-const wird einmal deklariert und ist nachher unveränderlich. Bspw. PI wäre ein Beispiel für eine typische const.

Um zu überprüfen ob das Programm auch wirklich das macht, was der Entwickler erwartet, lohnt es sich Zwischenwerte zu speichern und zu überprüfen. Mit **console.log** oder window.**alert** können diese Werte entweder in der Konsole oder auch direkt auf der Webseite angezeigt werden.

Arrow-Functions: 
```js
**function(t){ return t * t}**  
```
wird zu
```js
**t => t*t**
```
Erspart Aufwand und Zeit.
Ebenso Hilfreich um Werte aus Konstruktoren zu entnehmen.

<CustomImage src="https://raw.githubusercontent.com/AndrinTraeger/blogposts/main/images/ts.png" alt="TypeScript" />

**Leitfragen**

**Notizen**
Typescript gibt Javascript das, was es nicht hat. Eine Spezifikation für Datentypen und Objektorientierung. 

Es wird dem Entwickler ermöglich, Klassen, Interfaces … zu erstellen und mittels Vererbung zu verknüpfen. Dazu können einzelne Variable wie auch Funktion private, public, protected gesetzt werden. 

Private: Man kann nur in der Klasse darauf zugreifen.
Public: Für alle zugreifbar.
Protected: Nur in der eigenen Klasse und in den Klassen die erben zugreifbar.
Protected, Public, Private = Modificators

Durch die Typenspezifizierung entstehen wenigere Probleme. Eine var: number kann nicht im Nachhinein durch einen String ersetzt werden.
Daher dass solche Probleme gerne einmal entstehen, wenn mehrere Personen an einem Projekt arbeiten, ist Typescript optimal für Teams in Betrieben.

Die Objektorientierung ist das faszinierendste an Typescript. Mit Klassen und Interfaces zu arbeiten, erbringt einen besseren Überblick über das Programm.

Mit Generics kann man Klassen und Methoden mit Typparametern parametrisieren. Damit wird die Typsicherheit trotz generischer Programmierung ermöglicht.

Von abstrakten Klassen kann kein Objekt mehr instanziiert werden, sondern dass ganze wird an die Unterklassen weitergegeben.

Kommandozeile:
Mit **tsc -w** wird das .ts Dokument, das geöffnet wird automatisch bei jeder Änderung kompiliert.
Mit **tsc {dateiname}** wird das .ts Dokument manuell kompiliert.

React ist eine auf JavaScript-basierende UI-Entwicklungsbibliothek. 

**useState:** 

Erlaubte es eine state zu einer Komponente hinzuzufügen

**useEffect:**

Ermöglicht es , Side Effects in einer Komponente auszuführen. Einige Beispiele für Side Effects sind das Abrufen von Daten, das direkte Aktualisieren des DOM und Zeitgeber. 
Es akzeptiert zwei Argumente.

**useRef:**

Der useRef Hook bietet eine unkomplizierte Möglichkeit, einen Verweis auf ein DOM-Element zu erstellen, sodass es im Kontext der Komponente leicht zugänglich und manipulierbar ist.

**useReducer:** 

dient dazu, aus dem aktuellen State und der jeweiligen Action mit ihren type - und payload -Eigenschaften einen neuen State zu erzeugen.

**useCallback:** 

Gibt eine zwischengespeicherte Version zurück, es sei denn, eine ihrer Abhängigkeiten z. B. State oder Props ändert sich

**useMemo:** 

ermöglicht es uns, eine Komponente zu merken.

**useTransition:** 

ist ein React Hook, mit dem der Status aktualisiert werden kann, ohne die Benutzeroberfläche zu blockieren.

Komponenten sind voneinander unabhängige Funktionselemente, die in einer Anwendung wiederverwendet werden können, und sie stellen die Grundbausteine aller React-Anwendungen dar.

Die drei Hooks useCallback, useMemo & useTransition sind relevant um die Performance einer Website zu verbessern. Memo() kann teilweise auch verwendet werden.

**useCallback:** 

Die Hook gibt eine gespeicherte Version der Funktion zurück, die sich nur ändert, wenn sich einer der Werte im Dependency-Array ändert. Durch die Speicherung der Funktion können unnötige Re-renderings vermieden und die Leistung Ihrer React-Anwendung verbessert werden.

**useMemo:** 

Die useMemo-Hook ist ein Hook in React, mit der ein Wert gespeichert werden kann. Er nimmt zwei Argumente entgegen: Das erste ist eine Funktion, die den Wert zurückgibt, der gespeichert werden möchte, und das zweite ist ein Dependency-Array. Die Hook gibt den gespeicherten Wert zurück, der sich nur ändert, wenn sich einer der Werte im Dependency-Array ändert.

**useTransition:** 

Ermöglicht es eine State zu updaten ohne die UI zu blockieren. Stattdessen wird ein Loading Screen eingeblendet.
